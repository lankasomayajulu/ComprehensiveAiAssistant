- This will be your second task for the application.
- This application will implement a Kanban type custom application. The application is a comprehensive, testing, reporting, planning, implementing different kinds of projects.
- A Project can be of type "Story Creation", "Coding", "Audio Generation", "Image Generation", "Video Generation". Additional Project types can be added later.
- Only Coding projects will work on real file system. A Coding Project when created will ask for the project name, the root folder of the project, the project description.
- All Other projects will make use of Virtual Files and also some real files as necessary. For example, "Story Creation" will only have Virtual Files. Others will have actual files. I will talk about them in a while.
- Please take a look at the following.
- Each Project will automatically have a Kanban Board. In this board, I should be able to create Cards.
- The cards will go through the phases of "Not Started", "In Development", "Testing", "Review" and "Completed". For different phases, have a different color scheme and all the cards should actually show a border in that color.
- Each Card should have a component, which will be visible in the "Board Tab" when I click on the card, with a button to "Go Back" to the Board Page. Also all the cards should be draggable to different phases. I should be able to click on any part of the card and drag it to different phase.
- Each card should be a discussion thread, where I can discuss in the thread and other users can reply or there should be a button called "Generate AI Response"
- All the "AI Responses" will work based on the Project type and the files it is working with. They will make use of the MCP server to generate the response. For now all the AI should work with "Openrouter" using the user's Openrouter key.
- Any AI related work should be logged with the user who generated the request, the input tokens, the output tokens, the model used, the date and time it was requested, the cost of the AI request, the finish reason.
- I should be able to assign other users to a project and they will be able to see that project and associated Kanban Board.
- Access to a project should be decided by either the user creating the project or admin.
- For example, if I create a Project, then in the Project Settings, I should be able to add contributing users and they should be able to see the Project, its cards/tickets in the Kanban board, They should be able to add Cards, move cards, etc.
- The Project Settings should include "Default Openrouter Model".
- A Project can be created in the dashboard page, and user should be able to see the project page in a card. The dashboard will have a lot of cards, for now let there be a large card, occupying half the width on the dashboard page. Let there be 2 more dummy cards, which occupy the remaining dashboard. The remaining cards should appear one below the other, while the one main card should be at the left and should occupy half the width and also the entire height of the content area.
- The left card should be a carousal style. The Carousal items should be manually scrollable. The first Carousal Item should be list of Projects, with the project name (Which can be trimmed to see that it does not cross the region, the author, the date it was created, number of contributing users.
- When I click on the Project Page, it should take me to the Project Page, and there you should see 3 tabs. The first tab is the Kanban Board, the second tab should be the Play Area, the third tab should be logs.
